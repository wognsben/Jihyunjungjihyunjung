결론

맞는 말 같아.
특히 네 경우엔 y만 제거하고 opacity + 0.8s + easing은 유지하는 게 가장 합리적인 1차 수정이야.

왜 맞냐

내가 앞서 말한 핵심은 사실 이것이었어:

“스크롤이 이동하는 느낌”의 주원인은 y: 20 -> 0

일반 이동과 BACK 복원은 분리해서 봐야 함

BACK 복원 로직은 유지해야 함

피그마도 이 부분은 정확히 잡았고,
추가로 너의 디자인 컨셉까지 반영해서 더 좋은 처방으로 좁힌 거야.

피그마 말 중 핵심적으로 맞는 부분
1. y와 duration은 별개

이거 맞아.

내가 “duration도 줄이면 더 즉각적”이라고 한 건 UX 우선 기준의 제안이었고,
네 프로젝트처럼 아트 포트폴리오 / 갤러리 미학이 중요한 경우엔
반드시 duration을 줄여야 하는 건 아니야.

즉:

문제 원인 = y

디자인 톤 = 0.8s fade

라면,
y만 제거하고 fade는 유지하는 게 훨씬 맞다.

2. useLayoutEffect로 scrollTo가 이미 paint 전에 처리된다

이것도 맞는 포인트야.

현재 구조가 정말 그 흐름대로 되어 있다면, 새 페이지 mount 시:

opacity 0 상태

이미 scroll은 top으로 세팅

그 다음 fade-in

이니까, 사용자는 스크롤이 움직이는 장면을 보지 않고
그냥 제자리에서 페이지가 나타나는 것처럼 느끼게 돼.

즉, 추가로 hashchange 직후 scrollTo를 넣는 식의 개입은 오히려 역효과일 수 있어.

3. BACK 복원엔 영향이 거의 없음

이것도 맞아.

y 제거는 시각적 전환 레이어만 바꾸는 거고,
실제 scroll 저장/복원 로직은 손대지 않으니까
현재 BACK 복원 동작을 유지하기에 가장 안전한 수정이야.

그래서 지금 가장 좋은 판단
우선순위 1

정말 딱 이것만 먼저 바꾸는 게 맞아.

기존:

initial={{ opacity: 0, y: 20 }}
animate={{ opacity: 1, y: 0 }}
exit={{ opacity: 0, y: -20 }}

수정:

initial={{ opacity: 0 }}
animate={{ opacity: 1 }}
exit={{ opacity: 0 }}

그리고 transition은 그대로:

transition={{ duration: 0.8, ease: [0.16, 1, 0.3, 1] }}

이렇게.

이 수정으로 기대되는 결과

일반 페이지 이동 시
→ “아래에서 위로 밀려 올라오는 느낌” 제거

새 페이지가
→ 정지된 상태에서 부드럽게 나타남

갤러리/포트폴리오 무드 유지

BACK 복원 로직 손상 없음

코드 변경도 매우 작음

내가 보기에 피그마 답변에서 제일 좋은 점

“진단은 유지하고, 처방은 프로젝트 톤에 맞게 최소 수정으로 제한했다”는 점이야.

이건 실무적으로 되게 좋은 접근이야.
괜히 여러 군데 건드리면:

디자인 톤 깨지고

복원 로직 흔들리고

다른 버그 생기고

그럴 수 있는데, 이번 건은 원인과 수정 지점이 거의 1:1로 연결돼 있어.
