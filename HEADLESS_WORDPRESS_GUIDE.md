# Headless WordPress 구조 설명

## 현재 상황
- ✅ **WordPress 백엔드**: 데이터 관리용 (블로그 형태로 보임)
- ✅ **피그마 메이크 프론트엔드**: 실제 홈페이지 디자인

네, 맞습니다. **Headless WordPress** 방식으로 진행하면 됩니다.

## 개념 설명

### Headless WordPress란?
WordPress는 백엔드(데이터 관리)만 사용하고, 프론트엔드는 별도로 제작하는 방식입니다.

```text
┌─────────────────┐         ┌──────────────────┐
│  WordPress      │         │  프론트엔드      │
│  (백엔드)       │  API    │  (피그마 메이크) │
│  - 데이터 관리  │ ←──────→│  - 실제 홈페이지 │
│  - 관리자 페이지│         │  - 디자인        │
└─────────────────┘         └──────────────────┘
```

## 실제 구조

### 1. WordPress (백엔드) - 현재 보이는 블로그 형태
- **URL**: `https://wognsben97.mycafe24.com`
- **용도**:
  - **관리자 페이지**: `https://wognsben97.mycafe24.com/wp-admin/`
  - 클라이언트가 글 작성/수정하는 곳
  - **API 제공**: `https://wognsben97.mycafe24.com/wp-json/wp/v2/`
- **사용자**: 관리자, 클라이언트
- **참고**: 블로그처럼 보여도 됩니다. 관리자만 사용합니다.

### 2. 프론트엔드 (피그마 메이크) - 실제 홈페이지
- **URL**: 실제 홈페이지 도메인 (나중에 클라이언트 도메인으로 변경)
- **용도**:
  - 실제 사용자가 보는 홈페이지
  - WordPress API로 데이터 가져오기
- **사용자**: 일반 방문자

## 작동 흐름

### 1. 클라이언트가 글 작성
1. 클라이언트가 `https://wognsben97.mycafe24.com/wp-admin/` 접속
2. WordPress 관리자 페이지에서 글 작성
3. 저장

### 2. 방문자가 홈페이지 접속
1. 방문자가 `https://jihyunjung.com` (프론트엔드) 접속
2. 프론트엔드가 WordPress API 호출
3. `https://wognsben97.mycafe24.com/wp-json/wp/v2/work` 에서 데이터 수신
4. 화면에 데이터 표시

## 요약

**질문**: WordPress가 블로그처럼 보이는데, 홈페이지를 따로 만들어야 하나요?

**답변**: 네, 맞습니다.

1. **WordPress (백엔드)**: 데이터 관리용 (블로그처럼 보여도 괜찮음)
   - 관리자만 사용
   - 클라이언트가 글 작성하는 곳

2. **프론트엔드 (피그마 메이크)**: 실제 홈페이지
   - 일반 사용자가 보는 곳
   - WordPress API로 데이터 가져오기

이것이 **Headless WordPress** 방식입니다.
